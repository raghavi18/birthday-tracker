Auto-Backup Status

Every change is mirrored to a timestamped JSON file on the server's /backups directory and the most recent snapshot is overwritten as latest.json. If the live database is ever lost, the API auto-restores from the latest backup at startup.

Restoring From Backup

To restore a previous snapshot manually:

  1. Stop the backend service.
  2. Replace backups/latest.json with the snapshot you want to restore from.
  3. Delete or rename the existing birthdays.db file.
  4. Start the backend — it will detect the missing database and rebuild it from latest.json.
